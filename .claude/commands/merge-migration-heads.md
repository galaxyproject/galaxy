Resolve multiple Alembic migration heads in the Galaxy database schema.

## Context

Galaxy uses Alembic for database schema migrations with two branches:
- `gxy` (galaxy model) in `lib/galaxy/model/migrations/alembic/versions_gxy/`
- `tsi` (tool shed install) in `lib/galaxy/model/migrations/alembic/versions_tsi/`

Multiple heads occur when diverged branches (feature branches, release merges) each add
migrations descending from the same parent. Alembic requires a single head per branch.

## Steps

1. **Detect heads** — run `sh scripts/run_alembic.sh heads` from the repo root. Report
   which branch(es) have multiple heads and list the revision IDs.

2. **Identify the divergence** — for each branch with multiple heads, read the migration
   files for those head revisions in the appropriate `versions_gxy/` or `versions_tsi/`
   directory. Determine:
   - What each head migration does (the `upgrade()` function)
   - Their common ancestor (`down_revision` chains)
   - Whether the migrations have conflicting schema changes (e.g. both alter the same table/column)

3. **Check for conflicts** — if both heads modify the same table or column, flag this to
   the user and explain the conflict before proceeding. If they're independent schema
   changes (different tables), note that the merge is safe.

4. **Create the merge migration** — run:
   ```
   sh scripts/run_alembic.sh merge -m "Merge migration heads" heads
   ```
   This creates a new file in the appropriate `versions_*/` directory with empty
   `upgrade()`/`downgrade()` and a tuple `down_revision`.

5. **Verify** — run `sh scripts/run_alembic.sh heads` again and confirm each branch
   now has exactly one head.

6. **Report** — show the user the created merge file path and explain what was merged.

## Notes

- Merge migrations have empty `upgrade()`/`downgrade()` — they're purely structural.
- The `down_revision` in a merge migration is a tuple: `("rev_a", "rev_b")`.
- Naming convention: `<12-char-hex>_<snake_case_description>.py` (auto-generated by Alembic).
- Example existing merge: `lib/galaxy/model/migrations/alembic/versions_gxy/460d0ecd1dd8_.py`.
- If `scripts/run_alembic.sh` isn't available, fall back to: `cd lib/galaxy/model/migrations && alembic heads`.
